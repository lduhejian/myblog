基础概念

c 语言的数组是一块可以被偏移量 offset(offset 是一种 key) 访问的内存区。这也就暗含着这些 key 必须是整数，并且是连续的。例如，你知道数组的前三个索引是 0, 1, 2, 那么他的下一个索引必须是 4, 不能是 214678462。相比之下，php 的数组结构就非常不同：他既支持字符串类型的 key, 也支持非连续的整数做为 key, 甚至两种还可以一起使用在一个数组中。

c 语言可以有两种方式实现这种 php 的数组结构。一种用二叉搜索树，使用这种实现方式，无论是查找还是插入都是 O(log n) 的时间复杂度(n 是表中的元素数量) 。第二种是使用 hashtable 实现，这种实现方式的查找和插入的时间复杂度的平均值是 O(1), 也就是说，元素可以一很短的时间插入和获取。因此在多数情况下，hashtable 更被人喜欢， php 数组的实现也是用的这种技术。

hashtable 的原理是非常简单的：复杂的 key(例如 string) 被 hash 函数转换成整数。这个整数可以用做 c 数组的 offset。 问题是，无论是 2^32 还是 2^64 都要比字符串的数量少（因为他们是无尽的）。因此 hash 函数会有冲突，也就是说两个不同的 key, 会得到相同的 hash 值。

因此这种冲突的情况肯定会发生。对于这个问题有两个基础的解决办法，一个是 open addressing(在这里我们不讨论)。另一个是链表，这是 php 选择的方式。这种方式简单的把有相同 hash 值的所有元素放在一个链表中。当我们通过 key 来对值进行访问时，php 会先计算 key 的 hash 值，然后遍历链表，直到发现匹配的 entry。下面是一个链表冲突的分析。

链表中的元素叫做 bucket, C 数组中包含了链表的 head。c 数组叫 arBuckets。

考虑到你需要删除一个指定的 bucket: 例如你有一个指向 bucket "c" 的指针，你想删除 "c"。为了实现这样的目的，你需要移动 bucket "a" 的指针指向 NULL。因此你需要获得 bucket "a", 实现的方式有两种，一种是通过 hash 值遍历链表，还有一种是为每个 bucket 再增加一个相反方向的指针（变成双向链表）。php 采用了第二种实现方式：每一个 bucket 包含一个指向下一个 bucket 的 pointer (pNext），和一个指向前一个 bucket 的指针(pLast)。下面是这种结构的图解：

HashTable 和 bucket 数据结构(structure)

为了实现 hashtable php 使用了两种数据结构，可以在 zend_hash.h 中找到。我们首先看一下 bucket 的数据结构:

```c
typedef struct bucket {
    ulong h;
    uint nKeyLength;
    void *pData;
    void *pDataPtr;
    struct bucket *pListNext;
    struct bucket *pListLast;
    struct bucket *pNext;
    struct bucket *pLast;
    char *arKey;
} Bucket;
```

我们前面已经知道了 pNext, pLast, pListNext 和 pListLast 作用。现在让我们快速的过一下其他没讲到的属性（member）:　
h 是 bucket 对应的 hash key. 如果 key 是整数，h = key (hash function 不对整数进行映射), nKeyLength 将会赋值为 0. 如果 key 是字符串，h = zend_hash_func(), arKey = &key, nKeyLength 是字符串的长度。

pData 是指向储存值的指针。存储的值和通过 insertion function 传入的值不是一个，他是传入值的 copy（没有存储在 bucket 中）。因为以指针的方式指向存储数据是非常没有效率的，php 使用了一个小的技巧来改善：...

现在让我们来看一下最主要的 HashTable 结构:
```c
typedef struct _hashtable {
    uint nTableSize;
    uint nTableMask;
    uint nNumOfElements;
    ulong nNextFreeElement;
    Bucket *pInternalPointer;
    Bucket *pListHead;
    Bucket *pListTail;
    Bucket **arBuckets;
    dtor_func_t pDestructor;
    zend_bool persistent;
    unsigned char nApplyCount;
    zend_bool bApplyProtection;
#if ZEND_DEBUG
    int inconsistent;
#endif
} HashTable;
```

你已经知道 arrBuckets 是一个 c 数组，他包含了许多链表并且他里面装的是 key 的 hash 值。因为 php 的数组没有固定的大小，所以当元素的数量超过 arrBuckets 分配（内存）数量的时候, arBuckets  也必须增加内存分配。我们当然可以在 hashtable 中存储超过 nTableSize 的元素，但是这将会增大 hash 冲突的数量，降低数组的性能表现。

nTableSize 总是 2 的幂次方，因此如果你的 hashtable 中有 12 个元素，nTableSize 的值将会是 16.注意 arBuckets 会随着元素的增多自动增长，但是当你移除元素的时候，他并不会减少。如果你在 hashtable 中增加 1000000 个元素，接着再将它们移除，nTableSize 的值仍将会是 1048576.

hash 函数的计算结果是 ulong 类型，但是 nTableSize 通常会比这个值小一点。因此 hash 值不能直接用作 arBuckets 的 index。
